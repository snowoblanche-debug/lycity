# LY.city V3.0 Architecture Approval

After reviewing the proposed architecture, I approve the overall direction with the following decisions and priorities.

---

# 1. Multi-Tenant Architecture

Choose:

## Option A — Complete Isolation (Approved)

Each streamer should have their own:

* Song Library
* Queue
* Song History
* Statistics
* OBS Settings
* Lyrics
* Categories
* Overlay Settings

Future URLs:

https://ly.city/oriyuki

https://ly.city/shiro

https://ly.city/tako

This is the correct long-term architecture.

However:

Please keep Collection Sharing as a future feature so streamers can selectively share song libraries without sharing their entire database.

---

# 2. V3 Phase 1 Scope

Please implement:

## P1 — Lyrics System

## P4 — Song Cooldown

## P5 — Request Notes

together in Phase 1.

Do NOT start OAuth, Collections, or advanced multi-streamer migration yet.

Focus on the actual singing workflow first.

---

# 3. Lyrics System Requirements

This is the highest priority feature.

The goal is:

Song
↓
Lyrics
↓
OBS
↓
Performance History

Complete the full workflow.

* Email login outage
* Database issues
* Emergency maintenance

Current login should continue working.
