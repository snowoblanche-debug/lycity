V2.3 UX & Information Architecture Improvements

The current version is functional, but there are still several UX issues that impact streamer usability.

Please implement the following improvements.

1. CATEGORY MANAGEMENT READABILITY

Current issue:

The Category Management page still has poor text contrast.

Examples:

名稱
類型
操作

and category names:

神曲
中文
日語
英語

blend into the background.

Readability is currently below WCAG AA.

Required Fix

Table Headers:

color: #243447;
font-weight: 700;

Table Content:

color: #374151;
font-weight: 500;

Category Type Badges:

color: #374151;
background: #F3F4F6;
border: 1px solid #D1D5DB;

Do NOT use:

opacity: 0.5;
opacity: 0.6;

for text.

Text should remain readable even on bright displays.

2. OBS URL SHOULD USE SITE BRANDING

Current issue:

OBS URL displays internal Replit hostname.

Example:

https://87356162-b589-4487-xxxx.replit.dev/obs

This is confusing for streamers.

Required Fix

OBS URL should use the configured public site URL.

Add new setting:

Site URL

Example:

https://ly.city

OBS URL preview should become:

Public:

https://ly.city/obs

Protected:

https://ly.city/obs?key=xxxxx


Logic:

```text
settings.site_url
+
/obs

instead of:

window.location.origin
3. PRIMARY TAG / SECONDARY TAG SYSTEM

Current song tagging system is flat.

All tags are treated equally.

This becomes difficult to manage when the library grows to 300~1000 songs.

Database Changes

Songs table:

primary_tag_id

New relationship:

Song
├── Primary Tag (1)
└── Secondary Tags (many)

Example:

Song:

受協

Primary Tag:

抒情

Secondary Tags:

EMO
神曲
