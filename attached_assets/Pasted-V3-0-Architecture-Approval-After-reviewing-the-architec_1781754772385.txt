V3.0 Architecture Approval

After reviewing the architecture proposal, I approve the following decisions.

Multi-Tenant Architecture

Approved:

Option A — Complete Isolation

Each streamer should have their own:

Songs

Statistics

Future URLs:

Performance

Providers:

LrcLib (Primary)

Important Workflow Decision

Regarding:

started_at

I do NOT want automatic timestamps.

Please use explicit streamer actions.

Current workflow should become:

Waiting
↓
🎵 開始演唱
↓
Playing
↓
✅ 完唱
↓
History

Start Singing Button

Add:

🎵 開始演唱

in Queue Management.

When clicked:

status = playing
started_at = now
OBS updates
Lyrics overlay updates
Complete Song Button

When clicked:

status = completed
completed_at = now
write song_history
increment sing_count
update statistics
update cooldown tracking
Skip Song Button

When clicked:

status = skipped
no sing_count increment
no history record
no statistics update
